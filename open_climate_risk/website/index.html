Capabilities
Everything you need for
climate risk screening
From raw hazard data to boardroom-ready reports, in a reproducible Python workflow.

Hazard Extraction

Automatically sample WRI Aqueduct flood depth rasters for any coordinate. Supports both local GeoTIFFs and cloud-optimized streaming from S3.

Vulnerability Curves

JRC depth-damage functions (Huizinga et al. 2017) for residential, commercial, industrial, agriculture, and transport assets.

Expected Annual Damage

Trapezoidal integration over the full exceedance-probability curve. Express risk as EUR/year and % of asset value.

Climate Scenarios

Compare historical baseline vs. RCP 4.5 and RCP 8.5 at 2050. Ensemble of 5 ISIMIP2b GCMs with median and min/max spread.

Portfolio Analysis

Batch-process hundreds of assets. Map portfolio exposure, identify hotspots, rank by EAD, and compare coastal vs. riverine risk.

Automated Reports

Generate branded Word (.docx) and Excel (.xlsx) reports with all figures, tables, methodology notes, and data exports.

How it works
Four-stage risk pipeline
Each step is a composable Python function. Chain them in a notebook or script.
1

Define Asset

Location (lon/lat), type (residential, industrial...), replacement value, floor area.

2

Extract Hazard

Sample Aqueduct GeoTIFFs at 9 return periods. Riverine (5 GCMs) or coastal (4 scenarios).

3

Apply Damage

Interpolate JRC depth-damage curves. Convert flood depth to damage ratio and monetary loss.

4

Compute EAD

Integrate over the exceedance-probability curve. Compare scenarios, generate reports.

Package structure
Clean, modular architecture
Five focused modules, each with a clear responsibility.
open_climate_risk/
  pyproject.toml # metadata & deps
  src/open_climate_risk/
    config.py # paths, GCMs, scenarios
    data.py # raster I/O, JRC curves
    analysis.py # EAD, damage, ensemble
    plot.py # all matplotlib figures
    report.py # Word & Excel generation
  notebooks/
    01_riverine_flood_risk.ipynb
    02_coastal_flood_risk.ipynb
  data/aqueduct/ # 135 GeoTIFFs
  references/ # JRC curves
  tests/ # 22 unit tests

config.py

Project paths, Aqueduct S3 URLs, return periods, GCM model definitions, climate scenario configurations for riverine and coastal.

data.py

Raster sampling (local or S3 streaming), JRC vulnerability curve loading, batch extraction for portfolios, coastal source construction.

analysis.py

Depth-damage application, EAD computation via trapezoidal integration, multi-GCM ensemble extraction, portfolio batch processing.

plot.py

Risk screening dashboard, hazard curves (per GCM / per scenario), portfolio maps, scenario comparisons. Publication-quality matplotlib.

report.py

Automated Word report with executive summary, methodology, figures, and data tables. Excel export with styled formatting.

Roadmap
What's coming next
Building toward a comprehensive open-source physical risk platform.
Now — Q1 2026

Riverine + coastal flood modules. Single-asset and portfolio analysis. JRC vulnerability curves. Automated Word/Excel reports. 22 unit tests.

API & Financial Metrics

FastAPI + PostGIS geospatial API. Financial risk metrics: NPV of expected losses, Value-at-Risk (VaR), Marginal Expected Shortfall. TCFD-aligned reporting.
